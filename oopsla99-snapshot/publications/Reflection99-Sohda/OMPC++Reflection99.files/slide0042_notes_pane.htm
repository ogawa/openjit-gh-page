OpenJIT is a JIT compiler for Java which is written in Java and which can be customize dynamically using Reflection and Open Compiler.
In OpenJIT, users can customize compile operation of JIT. So we can apply optimization specialized for a certain application or some runtime enviroment. OpenJIT can also be used for extending Java.
Despite being reflective, OpenJIT is the second fastest JIT compiler on Linux, and will be released later this summer.

What is the present most portable language? Probably, it is Java language. However, this Java language also has only the portability which JavaVM offers. That is, functional extensions or the optimization suitable for runtime environment cannot be performed.
Then, we are developing OpenJIT which is the system which applied the technique of reflection and open compiler to Java. OpenJIT is the project which we TIT and Fujitsu are developing. OpenJIT is, simply saying, the JIT compiler of Java which was written in Java and which can be customize dynamically.
OpenJIT is based on many researches about JIT including SUIF. The first release is already due to be performed soon.
In OpenJIT, a user can customize compile operation of JIT. The class which performs the optimization at the time of compile is dynamically read at the time of compile. We say this class compilets. The compilets changes processing of usual JIT compile and performs the optimization to which it specialized in application and runtime environment.
