We Compare our system with other systems about the portability.
The software DSM of the traditional page-based like TreadMarks are low portability because of OS dependent. For example, the trap of the page fault must be carried out etc..
The class-based software DSM like midway, can give a certain amount of portability. However, a user has to manually insert all memory operations carefully.
Program transformation systems using a macro or preprocessor are portable. However, the support of the full set of language and implementing of the system itself is difficult.
On the other hand, our system is easy to implementation. Moreover, our system is simple for a user to use. Our system gives still high platform portability and efficient DSM.
