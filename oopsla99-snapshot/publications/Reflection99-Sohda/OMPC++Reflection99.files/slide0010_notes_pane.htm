We describe the implementation of software DSM. This DSM implementation adopts sharing array. So we defined distributed shared array class as a template class. Taking such a way, compared with page-based DSM, it is advantageous in respect of flexibility and portability. The Write-Invalidate style is  adopted as a memory consistency protocol.
Distributed shared array class includes contents of array and management table. Array is divided into blocks. Block is the unit of memory transfer. The size of the block is BlockSize.
Management table has pointer to the real memory and the share status Åc
To access to memory from program, we overrides [](bracket) and the ()(parentheses) operators. The misscheck of memory access is performed in these operators.
Moreover, at the time of write access, you have to insert WriteStart() and WriteEnd() function before and after the access. Because it must have the ownership of the block for writing.
For reduction of the overhead of memory access, Invalidation is asynchronously invoked, methods are inlined.
