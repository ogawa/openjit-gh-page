We show the result of performance evaluation of OMPC++ system.
We investigated the throughput of read/write access as a basic DSM performance. Here, we changed various BlockSize and saw the influence. In all accesses, although the copy of memory is occurred, but no invalidation are invoked.
Next, we take up CG Kernel and FFT and LU kernel from SPLASH2 benchmarks generally used as performance evaluation of DSM system as examples of real applications.
We needed to be corrected with small in order to run SPLASH2 by the OMPC++ system. The main correction parts are declaration statement etc. and there is no change of the core program. Needed correction for each of FFT and LU was about 70 lines to the program of about 1000lines.
