Here we show a related research.
Shasta transforms transforms SMP-based programs into DSM-based programs for Alpha custer environment.
Shasta performs program transformation in binary level. Many low level optimizations are applied, for example, the check of memory access is pressed down to the average of 2 operations. Its memory access overhead is very small. Furthermore, since Shasta implementation is specialized in the Alpha memory channel, the high performance is attained. On the other hand, Shasta is less portable.
The Performance of Our system is not comparable to that of Shasta.
However, employing optimization techniques used in shasta, the performance improvement can be expected. As I show in the result of LU, the bunching of access checks is very effective. Furthermore, our system has higher portability.

There is Shasta as a system to which program transformation is performed and a shared memory program is executed in a distributed memory environment.
