Although there are some systems to realize platform portability, but each of them holds problems.
First, in the parallel programming languages like HPF, they archive high performance, but programming models will be restricted.
Automatic parallelizing compilers such as those for FORTRAN are difficult to port the compiler itself and there are often performance problems.
Although messaging libraries such as MPI and PVM offer portability up to certain degree, programming models are restricted and programming becomes complicated.

That is, by the current solutions, it is difficult to realize platform portability.
So what kind of solution can we take for the platform portability?
Java is portable language, but Java can work only on shared memory machine. Moreover, Java has only the portability which JavaVM offers. That is, functional extensions or the optimization suitable for runtime environment cannot be performed
